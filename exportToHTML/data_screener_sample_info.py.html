2    '''++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++ 
3       This script is modified from Tian Han's data_screener.py script. It has been adapted to 
4       check sample location information for the ARIS Geochem data compilation project. 
5        
6       This script is coded to identify and report problems found in the staged geochem sample 
7       location data in xlsx files before loading them loaded into the ARIS Geochem database. 
8       Please beware that this script only identifies and reports problems. The problems identified 
9       need to be fixed manually. 
10       
11      The file names for the xlsx sheets are used to code the ARIS numbers into the database, so 
12      the names are important. All files should be name XXXXX_loc.xlsx where XXXXX is the ARIS 
13      report number. 
14       
15      Input 
16            1) Path to the directory where the staged xlsx files reside 
17            2) aris_geochem Database path (this is not currently necessary as of June 1, 2017 
18            but may be used at a later date). 
19            3) Some hardcoded parameters  
20             
21      Output 
22            Display identified problems 
23    
24      Operation note 
25            1) The script has many independent code-blocks targeting different types of errors as listed 
26               below. Though each code-block can be executed randomly, it is strongly recommended to 
27               run them sequentially in the given order, one at a time. For example, it should start with 
28               checking format related problems in the xlsx files by commenting out all other code-blocks 
29               which are for checking other types of errors. Once the format-related errors are identified 
30               and fixed, the format-checking code-block should be commented out and then move on to the 
31               "analyte name checking block" ... 
32    
33            2) Duplicate samples should be given different "sample_code". Re-published samples should 
34               have the same 'sample_code' across all publications. Sections 15 and 16 are designed to 
35               ensure if the above requirements are met. 
36                
37       This script is able to check the following: 
38            1)  Check the format of the spreadsheet to ensure that all necessary columns are present, properly named and in 
39               the right order. 
40            2) Check that x, y, z coordinates and epsg codes are present and are in fact numeric. Check that z values, when 
41               present are between 0 and 3000 metres. 
42            3) Check that values entered in the sample_type column match the pre-determined options. The script will  
43               automatically turn any upper case character to lower case. 
44            4)  Check that values in sample_subtype match accepted values. This column is case sensitive. 
45            5)  Check that values entered in coord_conf match accepted values. 
46            6)  Confirm that points actually plot within the province of BC by computing NAD83 Lat Long on the fly and 
47               comparing to a shapefile with the BC boundary. 
48           7)  Check that all dates under sample_date are proper dates. 
49             
50        
51     Status 
52         In Development 
53    
54     Future improvements 
55           The algorithms used in 16) and 17) above generate too many false alarms. They are not helpful 
56           enough to identify duplicate samples.

86   #Project input coordinates to NAD83 lat. and long (EPSG_SRID = 4269) 
87   #Syntax: project2nad83(float, float, int) returns [nad83_long, nad83_lat] 
88   def project2nad83 (x_coord, y_coord, source_epsg): 
89    
90       #Create a point geometry using the given coordinates 
91       point = ogr.Geometry(ogr.wkbPoint) 
92       point.AddPoint(x_coord, y_coord) 
93    
94       #Specify source spatial reference and projection 
95       source_ref = osr.SpatialReference() 
96       source_ref.ImportFromEPSG(source_epsg) 
97    
98       #Define target spatial reference and projection based on input 
99       target_ref = osr.SpatialReference() 
100      target_ref.ImportFromEPSG(4269) 
101           
102      #Perform transformation 
103      transform = osr.CoordinateTransformation(source_ref, target_ref) 
104      point.Transform(transform) 
105       
106      return [str(point.GetX(0)), str(point.GetY(0))]
